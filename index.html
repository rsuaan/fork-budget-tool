ℹ️ What This Tool Can and Cannot Tell You

This tool is fully manual — it calculates based on what you enter. Use the questions below to gather the right info from the customer.

FeatureDetectable from Twilio data?Notes
Media Streams Not detectable No stream SID or event in Voice Insights. Ask the customer how many streams and whether they use both_tracks or single track.
SIPREC Not detectable No SIPREC flag in Voice Insights. Ask the customer if SIPREC is active and which track mode (default = both_tracks = 2 forks).
Real-Time Transcription Not detectable No Transcription flag in Voice Insights. Ask the customer if <Transcription> is in their TwiML and which track mode.
AMD Partial postflight_answered_by in Voice Insights ClickHouse confirms AMD ran (machine_start, human, etc.). Does not confirm whether it was still active when streams were created.
Fork limit kill event Not detectable The platform emits no diagnostic when a fork is killed. Only symptom: stream-stopped fires immediately with no StreamError after a 200 from the REST API.
Questions to ask the customer:
  • Are you using Media Streams? If yes — how many, and are they both_tracks or single track?
  • Are you using SIPREC? If yes — default config (both_tracks) or single track?
  • Are you using <Transcription>? If yes — which track mode?
  • Is AMD (MachineDetection=Enable) active on this call?
  • Are streams being created at the start of the call, or after AMD completes?
